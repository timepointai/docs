# API Overview

All consumer traffic goes through the **API Gateway** at `api.timepointai.com`, which routes requests to backend services. Each service also has a direct subdomain for service-to-service calls.

## Architecture

```
Client (iPhone, Web App, MCP, etc.)
  │
  ▼
api.timepointai.com  ──>  API Gateway (stateless reverse proxy)
  │
  ├── /api/v1/clockchain/*  ──>  clockchain.timepointai.com
  ├── /api/v1/billing/*     ──>  billing.timepointai.com
  ├── /api/v1/* (all else)  ──>  flash.timepointai.com
  ├── /internal/*           ──>  403 BLOCKED
  └── /health               ──>  gateway health (native)
```

## Domain Map

| Domain | Service | Auth | Notes |
|--------|---------|------|-------|
| `api.timepointai.com` | API Gateway | Forwards auth headers | Consumer-facing entry point — routes to backends |
| `flash.timepointai.com` | Flash | `X-Service-Key` or JWT | Auth authority, timepoint generation, SSE streaming |
| `clockchain.timepointai.com` | Clockchain | None (public reads) / `X-Service-Key` (writes) | Temporal causal graph |
| `pro.timepointai.com` | Pro Cloud | `PRO_API_KEY` or Flash JWT | SNAG social simulations |
| `billing.timepointai.com` | Billing | `X-Service-Key` | Credit packs, subscriptions, Apple/Stripe verification |
| `mcp.timepointai.com` | MCP Server | MCP API Key | AI agent interface (planned) |

**Consumer apps** (iPhone, web app, external clients) should use `api.timepointai.com` — the gateway routes transparently.

**Service-to-service calls** should use direct subdomains (e.g. `flash.timepointai.com`) to skip the gateway hop.

## Quick Examples

### Clockchain — No Auth Needed

```bash
# Get graph stats
curl https://clockchain.timepointai.com/api/v1/stats

# Search moments
curl "https://clockchain.timepointai.com/api/v1/moments?q=rome&limit=5"
```

### Flash — Render a Moment

```bash
curl -X POST https://api.timepointai.com/api/v1/timepoints/generate/sync \
  -H "Content-Type: application/json" \
  -H "X-Service-Key: YOUR_KEY" \
  -d '{"query": "Moon landing, July 20 1969", "generate_image": true}'
```

## Response Format

All APIs return JSON. Clockchain public endpoints return paginated results:

```json
{
  "items": [...],
  "total": 769,
  "limit": 10,
  "offset": 0
}
```

## Rate Limits

| Tier | Limit | Applies To |
|------|-------|------------|
| Public | 60/min | Clockchain unauthenticated reads |
| Auth reads | 300/min | Authenticated GET requests |
| Auth writes | 30/min | POST/PUT/DELETE with service key |

Rate limit headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`

## Internal Routes

The gateway blocks all `/internal/*` paths — these are for direct service-to-service calls only (e.g. billing callbacks to Flash via Railway internal networking). Never route internal calls through the gateway.

## CORS

All backend services allow cross-origin requests from `*.timepointai.com` subdomains. Each service also accepts a `CORS_ORIGINS` env var for additional origins.

| Service | Allowed Origins | Methods |
|---------|----------------|---------|
| Flash | `timepointai.com`, `app.`, `pro.`, `clockchain.`, `flash.`, `timepoint.ai` | All |
| Clockchain | `timepointai.com`, `app.`, `api.`, `flash.`, `localhost:3000` | GET, OPTIONS |
| Pro Cloud | `timepointai.com`, `app.`, `api.`, `flash.`, `timepoint.ai`, `localhost:3000` | All |
