# Accessibility

Timepoint AI is committed to making our documentation and services accessible to all users. This site targets conformance with **WCAG 2.1 Level A**, the baseline tier of the Web Content Accessibility Guidelines.

## What we do

- **Text alternatives** — all diagrams and non-text content include plain-text descriptions
- **Keyboard navigation** — all interactive elements are reachable and operable via keyboard
- **Semantic structure** — pages use proper heading hierarchy, labeled tables, and landmark regions
- **Readable text** — content uses clear language, descriptive link text, and consistent navigation
- **No time limits** — no content requires timed interaction
- **No flashing content** — no elements flash more than three times per second

## Known limitations

- ASCII art diagrams in code blocks are supplemented with text descriptions but may still be difficult to parse with some screen readers
- Some third-party embedded content (e.g. external API responses in examples) may not meet all accessibility criteria

## Feedback

If you encounter an accessibility barrier on this site, please open an issue at [github.com/timepointai/docs](https://github.com/timepointai/docs/issues) with the subject "Accessibility" and we will address it.
